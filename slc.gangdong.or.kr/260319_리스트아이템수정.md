# 2026.03.19 리스트 아이템 디자인 고도화 수정 사항

이 문서는 백엔드 개발자가 리스트 아이템(`card_list2.html`)의 변화된 구조를 이해하고, 동적 데이터를 바인딩할 때 참고할 수 있도록 작성되었습니다.

## 1. HTML 구조 변경 사항 (card_list2.html)

### 카드의 전체적인 구성
각 카드(`.card_item`)는 크게 **[썸네일], [카드정보], [하단버튼]** 3단계 구조로 정립되었습니다.

1.  **썸네일 영역 (`.thumbnail`)**:
    - **상태 플래그 추가**: 썸네일 안에 `<span class="flag-702869e7"></span>` 요소가 추가되었습니다. 썸네일 우측 하단에 고정 배치됩니다.
    - **비활성화 처리**: 카드 마감 시 `.card_item`에 `disabled` 클래스를 추가하면 썸네일에 자동으로 마스크가 적용됩니다.

2.  **카드 정보 영역 (`.card_info`)**:
    - **제목(`.title`)**: 긴 제목은 자동으로 1줄 말줄임표 처리되도록 스타일이 보강되었습니다.
    - **세션 리스트(`.box-4457fbb2`)**: 각 카드마다 하위 세션 정보를 보여줄 수 있는 별도의 박스 영역이 추가되었습니다. (기존 summary 정보 외에 상세 세션 정보를 반복문으로 처리할 수 있는 구조)
    - **플래그(`flag-702869e7`)**: 각 세션의 접수 상태를 명확히 보여주기 위해 정보 텍스트 옆에 추가되었습니다.

3.  **하단 상태 버튼 바 (`.bottom_btn`)**:
    - 카드의 최하단에 상담 신청 혹은 접수 마감을 알리는 버튼형 바가 신설되었습니다.

## 2. 동적 처리를 위한 가이드 (클래스 및 상태)

백엔드에서 상태값에 따라 분기 처리해야 할 주요 클래스입니다.

| 요소 | 상태 | 적용할 클래스 | 비고 |
| :--- | :--- | :--- | :--- |
| **플래그 (Flag)** | 접수중 | `.flag-702869e7.type-1` | 텍스트는 CSS `content`로 자동 처리됨 |
| | 접수마감 | `.flag-702869e7.type-2` | |
| **하단 버튼 (Bar)** | 진행중 | `.bottom_btn.status_active` | 텍스트: '상담 신청' |
| | 마감 | `.bottom_btn.status_closed` | 텍스트: '접수 마감' |
| **전체 카드** | 마감/비활성 | `.card_item.disabled` | 썸네일에 50% 블랙 마스크 적용용 |

## 3. CSS 변경 요약 (card_list2.css)

백엔드 개발자가 레이아웃 정렬을 수동으로 맞추지 않아도 되도록 자동화된 정렬 로직이 포함되었습니다.

-   **Flexbox 정렬**: `.card_item`을 플렉스 박스로 설정하고 `min-height: 480px`를 부여하여, 세션 개수와 상관없이 모든 카드의 높이가 일정하게 유지되며 하단 버튼의 높이도 일치합니다.
-   **말줄임 처리**: 제목 영역에 `-webkit-line-clamp: 1`을 적용하여 1줄 이상 넘어가지 않도록 관리됩니다.
-   **썸네일 마스크**: `.disabled` 카드 내부의 썸네일(`::before`)에 블랙 투명 마스크를 적용하는 스타일이 추가되었습니다.
